08
Roadmap

Ce qui reste à faire

Le site est fonctionnel et déployé. Quelques optimisations identifiées pour les prochaines itérations.

  1. i.
    Checkout multi-coffrets Le backend Stripe traite actuellement un coffret par session. Pour un panier avec plusieurs coffrets différents, refactor de la fonction Cloudflare nécessaire pour passer un tableau de line_items à Stripe.
  2. ii.
    Lien "Sur-mesure" dans la nav La page coffrets-personnalises-entreprises est accessible via les CTAs internes mais absente de la nav top. Ajouter un 4e item de nav pour visibilité directe.
  3. iii.
    Quantité minimum cohérente Le tunnel UI accepte quantité ≥ 1. Le backend Stripe exige minimum 5. Aligner les deux validations pour éviter une erreur silencieuse au paiement.
  4. iv.
    Persistance du logo en panier Le fichier logo uploadé (PNG/PDF) n'est pas sauvegardé localement avec le coffret (limite localStorage). À chaque commande, l'utilisateur doit re-uploader. Solution : upload immédiat vers un service de stockage et persistance de l'URL.
  5. v.
    Newsletter footer Le formulaire newsletter dans le footer est cosmétique (submit handler retourne false). À brancher sur un service email (Mailchimp, Brevo, Resend, etc.).
